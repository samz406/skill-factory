# Skill Factory 产品需求文档（PRD）


------

# 一、项目概述

## 1.1 项目背景

随着大语言模型（LLM）与 AI Agent 技术的发展，越来越多企业开始尝试通过 AI 提升客服、运营、销售、审核、知识管理等业务场景的效率。

当前企业 AI 落地过程中存在一个核心问题：

企业已经积累了大量业务知识与流程资产，但这些内容无法直接被 AI Agent 稳定执行。

企业已有资产包括：

- SOP（标准操作流程）
- FAQ
- 培训文档
- 流程说明
- 业务规则
- 历史案例
- 飞书/Confluence 文档
- 专家经验

而 AI Agent 实际运行所需要的是：

- 结构化规则
- 执行流程
- Prompt 指令
- Tool Calling 定义
- 约束条件
- 上下文描述
- 输出格式规范

当前大多数企业仍然依赖 Prompt 工程师手动编写 Agent 能力，存在构建门槛高、知识复用困难、维护成本高等问题。

因此，需要一套能够将企业知识自动转译为 AI Skill 的系统。

------

## 1.2 产品定位

Skill Factory 是一套面向企业 AI Agent 场景的 Skill 构建平台。

系统通过：

- 聊天交互
- 文档解析
- 图片理解
- AI 自动追问
- 结构化信息提取

将企业知识逐步转化为可执行 AI Skill。

产品核心定位：

> AI Agent 时代的知识编译器。

------

## 1.3 产品目标

本产品目标是帮助企业快速完成：

```text
企业知识
→
结构化 Skill
→
AI Agent 可执行能力
```

降低企业 AI 落地门槛。

------

# 二、核心问题分析

## 2.1 当前行业问题

当前企业 AI 落地主要存在以下问题：

| 问题                 | 描述                                 |
| -------------------- | ------------------------------------ |
| Prompt 编写门槛高    | 需要具备 Prompt Engineering 能力     |
| 企业文档无法直接使用 | SOP、FAQ 等无法直接转化为 Agent 能力 |
| 业务人员无法参与     | AI 构建主要依赖技术团队              |
| 缺少结构化能力层     | AI Agent 缺少稳定的执行规则          |
| 缺少统一 Skill 标准  | 不同团队能力难以复用                 |
| 缺少测试机制         | Skill 效果无法稳定验证               |

------

## 2.2 当前解决方案不足

目前市场上的 Agent Builder 平台主要聚焦：

- Workflow 编排
- Prompt 编辑
- Agent Runtime

但缺少：

```text
企业知识
→
Agent Skill
```

这一关键转译层。

------

# 三、产品设计理念

## 3.1 Chat-first Skill Builder

Skill Factory 不采用传统“上传文档 → 自动生成”的方式。

系统采用：

```text
聊天 + 文档 + 图片 + AI 追问
```

共同完成 Skill 构建。

系统本质上是：

> AI 协同式 Skill 构建助手。

------

## 3.2 人机协同生成

系统认为：

企业知识通常是不完整、不规范、非结构化的。

因此：

Skill 的生成过程必须包含：

- AI 自动提取
- 用户补充说明
- AI 自动追问
- 人工确认修正

而不是完全自动生成。

# 七、核心模块设计

# 7.1 Chat Builder

## 功能说明

Chat Builder 是系统核心入口。

负责通过聊天方式逐步收集：

- 业务规则
- 执行流程
- 工具依赖
- 输出规范
- 约束条件

并动态生成 Skill 草稿。

------

## 核心能力

| 能力       | 描述                  |
| ---------- | --------------------- |
| 意图理解   | 理解用户业务目标      |
| 信息提取   | 提取规则与流程        |
| 自动追问   | 发现缺失信息          |
| 上下文管理 | 维护 Skill 构建上下文 |
| 实时预览   | 动态生成 SkillSpec    |

------

# 7.5 SkillSpec Builder

系统不会直接生成 Prompt。

而是先生成统一中间结构：

# SkillSpec

------

## SkillSpec 示例

```json
{
  "name": "",
  "description": "",
  "role": "",
  "workflow": [],
  "rules": [],
  "tools": [],
  "constraints": [],
  "exceptions": [],
  "output_format": ""
}
```

------

## SkillSpec 作用

SkillSpec 是：

```text
AI Skill 的结构化能力描述
```

用于：

- 统一生成逻辑
- 支持多平台导出
- 方便测试与版本管理

------

# 7.7 Skill Tester

## 功能说明

用于模拟 Skill 执行效果。

用户输入测试问题后：

系统执行 Skill 并返回：

- AI 输出
- 规则检查
- 工具调用结果
- 效果评分

------

## 测试维度

| 指标            | 描述             |
| --------------- | ---------------- |
| 准确率          | 输出是否符合规则 |
| 合规性          | 是否违反约束     |
| 完整性          | 是否遗漏流程     |
| Tool 调用正确性 | API 调用是否正确 |
| 输出质量        | 回复是否符合要求 |
